## 1. INTRODUCTION

### 1.1 Purpose

This document defines the functional requirements to enhance the existing SAP 1099 MISC (NEC) extract program. The purpose of this enhancement is to improve accuracy, compliance, and usability of the 1099 MISC data output for Company Code 0001.

The enhancement introduces new logic and output formatting changes to ensure Accounting and Tax receive correct, reportable data for year-end processing.

## 3. SUPPORTING DOCUMENTS

### 3.1 As Is

The current 1099 Program pulls data from: BSEG, WITH_ITEM, BKPF, LFA1, ADRC.
All payments with withholding codes (QSSKZ) are included, including virtual cards (ZLSCH = 'B').
Output file stored in: `/usr/sap/interfaces/zfap/out` (e.g. `ZFGL_REUSE_1099_<timestamp>.DAT`).

### 3.2 To Be

- Add selection checkbox “Exclude Virtual Card Payments.”
- Exclude records meeting ZLSCH='B' AND QSSKZ≠blank.
- Enhance output with: split names, PO Box preference, email column, Excel download.
- No changes to variant name or directory path.

| Req ID | BRD ID | Requirement Name | Requirement Description |
| :--- | :--- | :--- | :--- |

| FR-02 | BR-02 | Recipient Name Logic | For Individuals (Vendor Type ‘S’): concatenate First, Middle, Last, and Suffix. For Companies (Vendor Type ‘T’): use LFA1-NAME1 and LFA1-NAME2. |

| FR-05 | BR-05 | Excel Download Feature | Display report in ALV Grid and enable Excel export after log generation. |

## APPENDIX A: KEY TERMS

| TERM | DEFINITION |
| :--- | :--- |
| 1099 MISC / 1099 NEC | IRS tax forms used to report payments made to vendors or non-employees. The 1099 MISC is used for miscellaneous payments; the 1099 NEC is used for reporting non-employee compensation. |
| 1099 Program | Custom SAP ABAP program responsible for extracting 1099 data for year-end reporting. Enhancement subject to this document. |
| Virtual Card Payment (ZLSCH = ‘B’) | Payment method used for credit card–based vendor disbursements. These are non-reportable for 1099 purposes and must be excluded. |
| BSEG | SAP Financial Accounting Line Item table. Stores accounting document details such as payment methods (ZLSCH) and withholding tax codes (QSSKZ). |
| WITH_ITEM | SAP table containing withholding tax information for line items, including taxable amounts and withholding codes. |
| BKPF | SAP Accounting Document Header table, containing document type, company code, posting date, and reference information. |
| LFA1 | Vendor Master (General Section) table, containing vendor names, account group, and address reference numbers. |
| ADRC | Central Address Management table storing address data such as PO Box, street, city, postal code, and region. |
| ADR6 | Table containing email addresses for business partners (vendors, customers, etc.). |
| ALV (ABAP List Viewer) | SAP grid display tool used for dynamic report visualization and export functionality (Excel download). |
| Selection Screen Checkbox (SEL_VIRT_EXCLUDE) | New program parameter allowing users to include or exclude Virtual Card payments (ZLSCH = ‘B’) from report results. |
